# TabletOrderApp

## Overview
**Menu.ca Orders** — KDS (Kitchen Display System) tablet app for restaurant order management.
- **Package:** `ca.menu.orders` | **Version:** 1.4.23 (versionCode 32)
- **Platform:** Android tablets (Samsung, landscape mode)
- **Stack:** React Native 0.81.5, Expo SDK 54, TypeScript, Zustand

## Architecture
- **API Server:** `https://orders.menu.ca` (Replit) — NOT Neon/Railway
- **Auth:** QR code pairing → device UUID + key → session token
- **Status flow:** `pending → confirmed → preparing → ready → completed`
- **Backward transitions:** Require `{ force: true }` in API body
- **Storage:** Supabase for logos/assets, AsyncStorage for local state

## Build Commands
```bash
# Install deps (if node_modules missing)
npm install

# Generate android/ (if missing or stale)
npx expo prebuild --platform android --clean

# Build APK (ALWAYS use these flags)
export NODE_OPTIONS="--max-old-space-size=4096"
export ANDROID_HOME=/opt/homebrew/share/android-commandlinetools
cd android && ./gradlew assembleRelease \
  -x lintVitalAnalyzeRelease \
  -x lintVitalReportRelease \
  -x lintVitalRelease
```
- **Expected build time:** 2-3 min warm, 3-5 min clean
- **Output:** `android/app/build/outputs/apk/release/app-release.apk` (~92MB)
- **DO NOT kill builds early**

## Upload to Supabase
```bash
SERVICE_KEY=$(op item get "Supabase - TabletOrderApp Releases Bucket" --fields password --reveal)
curl -X PUT "https://nthpbtdjhhnwfxqsxbvy.supabase.co/storage/v1/object/app-releases/MenuCaOrders-v${VERSION}.apk" \
  -H "Authorization: Bearer ${SERVICE_KEY}" \
  -H "Content-Type: application/vnd.android.package-archive" \
  --data-binary @"android/app/build/outputs/apk/release/app-release.apk"
```

## Sideload via ADB
```bash
adb devices  # verify tablet connected
adb install -r android/app/build/outputs/apk/release/app-release.apk
```

## Gotchas
- `/android` is gitignored — generated by `npx expo prebuild`
- Never delete `android/build` — only `android/app/build` is safe to clean
- Lint tasks WILL crash the build if not excluded with `-x` flags
- EAS cloud build artifacts expire after ~30 days — always upload to Supabase
- Accept button calls `onAccept` only (NOT onStatusChange) to avoid race condition
- Backward status transitions need `force: true` — backend rejects without it
- Play Store installs have different signing key — first sideload requires uninstall
- Print success modals removed — only show alerts on failure

## Version Bumping
Update THREE places:
1. `app.json` → `expo.version` + `expo.runtimeVersion`
2. `android/app/build.gradle` → `versionCode` (increment) + `versionName`
